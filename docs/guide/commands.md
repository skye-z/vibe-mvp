# 命令参考

VibeMVP 提供三个命令：

## mvp_init

使用自然语言需求初始化新项目。

**参数：**
- `projectName` (可选) - 项目名称
- `requirements` (必填) - 使用自然语言描述项目需求

**功能：**
- 创建 `.mvp/` 目录结构
- 生成模板文档（guide、modules、roadmap）
- 根据需求拆分 MVP 版本路线图

## mvp_add

向现有项目添加新需求。

**参数：**
- `requirements` (可选) - 使用自然语言描述新需求，不提供则自动分析

**功能：**
- 解析新需求
- 更新路线图，添加新版本
- 考虑前置条件和测试版本

## mvp_next

执行路线图中的下一个待处理任务。

**功能：**
- 从路线图中识别下一个任务
- 读取 guide 和 modules 了解需求细节
- 执行任务后更新文档并提交 git
